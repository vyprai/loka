# Sessions

A session is a single Firecracker microVM. It provides an isolated Linux environment with its own filesystem, network, and process tree.

## Lifecycle

Every session moves through these states:

| State | Description |
|---|---|
| `creating` | VM is booting from a snapshot or image |
| `provisioning` | Image is being pulled/converted (skipped if cached) |
| `running` | VM is ready to accept commands |
| `idle` | VM suspended; auto-wakes on any access (exec, shell, port-forward, sync, domain proxy) |
| `paused` | VM is suspended; memory is preserved on disk |
| `terminating` | VM is shutting down |
| `terminated` | VM is destroyed; resources are freed |

```
creating → provisioning → running ⇄ idle ⇄ paused → terminating → terminated
```

## Create a session



```python
from loka import LokaClient

client = LokaClient()
session = client.create_session(
    image="python:3.12-slim",
    mode="execute",
)
```


```typescript
import { LokaClient } from '@vypr-ai/loka-sdk';

const loka = new LokaClient();
const session = await loka.createSession({
    image: 'python:3.12-slim',
    mode: 'execute',
});
```


## Pause and resume

Pausing suspends the VM and writes memory to disk. Resuming restores it in milliseconds.



```python
client.pause_session(session.id)
# ... later
client.resume_session(session.id)
```


```typescript
await loka.pauseSession(session.id);
// ... later
await loka.resumeSession(session.id);
```


## Destroy a session



```python
client.destroy_session(session.id)
```


```typescript
await loka.destroySession(session.id);
```



## Port forwarding

Expose VM ports on your local machine. Ports can be declared at session creation or added on-demand to a running session.



```python
# Declare ports at creation
session = client.create_session(
    image="python:3.12-slim",
    ports=[{"local_port": 8080, "remote_port": 5000}],
)

# Add a port forward to a running session
client.port_forward(session.id, 8080, 5000)
```


```typescript
// Declare ports at creation
const session = await loka.createSession({
    image: 'python:3.12-slim',
    ports: [{ local_port: 8080, remote_port: 5000 }],
});

// Add a port forward to a running session
await session.portForward(8080, 5000);
```



CLI equivalent:

```bash
loka session create --image python:3.12-slim --port 8080:5000
loka session port-forward <id> 8080:5000
loka session ports <id>
```

## Local file mounting

Mount a local directory into a running session via a tunnel. Changes on the host are reflected inside the VM.

```bash
loka session mount <id> ./project /workspace
loka session mount <id> ~/data /data --read-only
```

## Storage mount sync

Synchronise data between a VM mount path and its backing cloud storage bucket.



```python
client.sync_mount(session.id, "/data", direction="push")
```


```typescript
await loka.syncMount(session.id, { mount_path: '/data', direction: 'push' });
```



CLI equivalent:

```bash
loka session sync <id> /data --direction push   # VM → bucket
loka session sync <id> /data --direction pull   # bucket → VM
```

## Interactive Shell

```bash
loka shell <id>
```

Opens a full PTY: tab completion, arrow keys, Ctrl-C, resize. Uses gRPC bidirectional streaming.

## Idle / Auto-wake

```bash
loka session idle <id>              # Suspend to save resources
loka exec <id> -- python3 script.py # Auto-wakes if idle
```

Sessions auto-wake on: exec, shell, sync, port-forward, domain proxy access.
Set `--idle-timeout 300` at create time for automatic idling after 5 min of inactivity.

## Domain Exposure

```bash
loka session expose <id> my-app --port 5000
# Access at: https://my-app.loka.example.com
loka session unexpose <id> my-app
loka domains
```

## Artifacts

Artifacts are files that have changed relative to the base image. They let you retrieve outputs (generated files, logs, trained models, etc.) from a session without needing a storage mount.

When a checkpoint is created, its artifact list is frozen at that point in time. You can list or download artifacts for the current session state or for any checkpoint.



```python
# List changed files
artifacts = client.list_artifacts(session.id)

# List artifacts for a specific checkpoint
artifacts = client.list_artifacts(session.id, checkpoint=cp.id)

# Download a single file
data = client.download_artifact(session.id, "/workspace/output.csv")

# Download all artifacts as a tar archive
client.download_artifacts(session.id, "./output/")
```


```typescript
// List changed files
const artifacts = await loka.listArtifacts(session.id);

// List artifacts for a specific checkpoint
const artifacts = await loka.listArtifacts(session.id, { checkpoint: cp.id });

// Download a single file
const data = await loka.downloadArtifact(session.id, '/workspace/output.csv');
```



CLI equivalent:

```bash
loka session artifacts <id>
loka session artifacts <id> --checkpoint <cpId>
loka session download <id> /workspace/output.csv
loka session download <id> --all ./output/
```

## Execution modes

Each session runs in a mode that controls what commands can do.

| Mode | Filesystem | Network | Use case |
|---|---|---|---|
| `inspect` | Read-only | Blocked | Safe code review |
| `plan` | Read-only | Blocked | Generate plans without side effects |
| `execute` | Read-write | Allowed | Run builds, tests, scripts |
| `commit` | Full access | Allowed | Deploy, push, write anywhere |
| `ask` | Read-write | Allowed | Like execute but every command needs approval |

See [Execution Modes](/concepts/modes.md) for details.
