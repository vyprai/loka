# Deploy to DigitalOcean

## Credentials

```bash
# Option 1: doctl CLI (recommended)
doctl auth init

# Option 2: Environment variable
export DIGITALOCEAN_TOKEN=dop_v1_...
```

## Deploy

```bash
loka deploy do --name prod --region nyc1 --workers 2
```

## Options

| Flag | Default | Description |
|------|---------|-------------|
| `--name` | `do` | Server name |
| `--region` | `nyc1` | Datacenter region |
| `--workers` | `1` | Number of worker Droplets |
| `--instance-type` | `s-8vcpu-16gb` | Droplet size |
| `--ssh-key` | | SSH key fingerprint or ID |

DigitalOcean Droplets support KVM natively — no special configuration needed.

## Scale

```bash
loka worker scale 4       # Scale to 4 workers
```

## Declarative

```yaml
name: prod
provider: do
region: nyc1
workers:
  - instance_type: s-8vcpu-16gb
  - instance_type: s-8vcpu-16gb
```

```bash
loka deploy apply prod.yml
```

## Manual setup

### 1. Create Droplets

```bash
doctl compute droplet create loka-cp \
  --image ubuntu-22-04-x64 --size s-4vcpu-8gb \
  --region nyc1 --ssh-keys <key-id>

doctl compute droplet create loka-worker-1 \
  --image ubuntu-22-04-x64 --size s-8vcpu-16gb \
  --region nyc1 --ssh-keys <key-id>
```

### 2. Install and start

```bash
ssh root@<ip>
curl -fsSL https://rizqme.github.io/loka/install.sh | bash

# On CP:
lokad --role controlplane

# On workers:
cat > /etc/loka/worker.yaml << YAML
control_plane:
  address: "<cp-ip>:6841"
token: "loka_..."
YAML
loka-worker
```

Or use SSH-based deployment:

```bash
loka deploy vm --name prod --cp <cp-ip> --worker <worker-ip>
```
