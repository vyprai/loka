# Filesystem Policy

Controls path-level access within the VM.

## Rule Structure

```json
{
  "filesystem_policy": {
    "rules": [
      {"path": "/app/**", "access": ["read", "write", "create", "list"]},
      {"path": "/tmp/**", "access": ["read", "write", "create", "delete"]},
      {"path": "/usr/bin/python3", "access": ["read", "run"]},
      {"path": "/etc/passwd", "access": ["read"]},
      {"path": "/proc/**", "access": ["read", "list"]},
      {"path": "/home/agent", "access": ["read", "write", "create", "list", "delete"]}
    ],
    "default_access": [],
    "mode_overrides": {
      "inspect": {"default_access": ["read", "list"]}
    }
  }
}
```

## Path Patterns

| Pattern | Example | Match |
|---|---|---|
| Exact | `/etc/passwd` | Single file |
| Directory | `/app/` | The directory itself |
| Glob | `/app/*.py` | Files matching glob in one level |
| Recursive | `/app/**` | All files and directories recursively |

## Access Types

| Type | Description |
|---|---|
| `read` | Read file contents |
| `write` | Modify existing files |
| `run` | Use as a command binary |
| `list` | List directory entries |
| `delete` | Remove files or directories |
| `create` | Create new files or directories |

## Mode Overrides

Mode overrides replace the `default_access` for a given session mode. Explicit rules always take precedence over the default.

Example: in `inspect` mode, the default access is read-only (`["read", "list"]`), but explicit rules granting write to `/tmp/**` still apply.

## Default Policy

When no filesystem policy is specified, the supervisor applies:

```json
{
  "rules": [
    {"path": "/home/agent/**", "access": ["read", "write", "create", "list", "delete"]},
    {"path": "/tmp/**", "access": ["read", "write", "create", "list", "delete"]},
    {"path": "/usr/**", "access": ["read", "run", "list"]},
    {"path": "/lib/**", "access": ["read", "list"]},
    {"path": "/proc/**", "access": ["read", "list"]},
    {"path": "/sys/**", "access": ["read", "list"]}
  ],
  "default_access": ["read"]
}
```

## Enforcement: FSGate

The filesystem policy is enforced by **FSGate**, which uses two mechanisms:

| Mechanism | Scope | Description |
|---|---|---|
| **Landlock** | Process-level | Linux Landlock LSM restricts filesystem access at the kernel level. Applied to the command process before it starts. |
| **seccomp-notify** | Syscall-level | Intercepts `open`, `openat`, `unlink`, `rename`, `mkdir`, `rmdir` syscalls. The supervisor inspects the path argument and allows or denies the call. |

Both layers are active simultaneously. A request must pass both to succeed.
