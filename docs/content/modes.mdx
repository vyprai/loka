# Execution Modes

Every session runs in a mode that controls filesystem access, network access, and whether commands need approval.

## Mode reference

| Mode | Filesystem | Network | Approval | Purpose |
|---|---|---|---|---|
| `inspect` | Read-only | Blocked | No | Read files, review code safely |
| `plan` | Read-only | Blocked | No | Generate plans without side effects |
| `execute` | Read-write | Allowed | No | Run builds, install packages, write files |
| `commit` | Full access | Allowed | No | Push to git, deploy, write to protected paths |
| `ask` | Read-write | Allowed | Required | Every command pauses for human/agent approval |

## What each mode allows

**inspect** -- The agent can read any file and run read-only commands. Writes are rejected at the filesystem layer. Network calls fail. Use this when you want the agent to look at code without changing anything.

**plan** -- Same restrictions as inspect. Intended for planning steps where the agent generates a list of actions to take later.

**execute** -- The agent can read and write files, install packages, and make network calls. This is the default for most tasks.

**commit** -- Unrestricted access. The agent can write to protected paths like `/etc` and push to remote repositories. Use this only for final deployment steps.

**ask** -- Same capabilities as execute, but every command enters a `pending_approval` state. A human or supervising agent must approve or reject each command before it runs.

## Switch modes at runtime



```python
from loka import LokaClient

client = LokaClient()
session = client.create_session(image="python:3.12-slim", mode="inspect")

# Agent finishes reviewing, ready to make changes
client.set_mode(session.id, "execute")

# Time to deploy
client.set_mode(session.id, "commit")
```


```typescript
import { LokaClient } from '@rizqme/loka-sdk';

const loka = new LokaClient();
const session = await loka.createSession({ image: 'python:3.12-slim', mode: 'inspect' });

// Agent finishes reviewing, ready to make changes
await loka.setMode(session.id, 'execute');

// Time to deploy
await loka.setMode(session.id, 'commit');
```



## Mode + command policy interaction

Modes work together with the [command policy](/security/command-policy.md). The command policy controls *which* binaries can run; the mode controls *what those binaries can do* (filesystem writes, network access).

A command must pass both checks:

1. The command policy must allow the binary (or route it to approval).
2. The mode must allow the operation the binary attempts (write, network call).
