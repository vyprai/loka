# Command Policy

A command policy defines which binaries a session can run and whether unknown commands need approval.

## How it works

Every command is evaluated against two lists:

| List | Effect |
|---|---|
| `allowed_commands` | Binary runs immediately, no approval needed |
| `blocked_commands` | Binary is rejected with an error |

If a binary is in neither list, it gets the `needs_approval` verdict. What happens next depends on the session's [execution mode](/concepts/modes.md).

## Three verdicts

| Verdict | Condition | Result |
|---|---|---|
| `allowed` | Binary is in `allowed_commands` | Runs immediately |
| `blocked` | Binary is in `blocked_commands` | Rejected with error |
| `needs_approval` | Binary is in neither list | Routed based on mode |

### Mode behavior for `needs_approval`

| Mode | Action |
|---|---|
| `execute` / `commit` | Command runs immediately |
| `ask` | Command enters `pending_approval` queue |
| `inspect` / `plan` | Command is rejected (read-only modes) |

## Evaluation order

1. Check `blocked_commands`. If matched, verdict is **blocked**.
2. Check `allowed_commands`. If matched, verdict is **allowed**.
3. Otherwise, verdict is **needs_approval**.

Blocked commands are checked first, so a binary in both lists is blocked.

## Create a session with a policy



```python
from loka import LokaClient

client = LokaClient()
session = client.create_session(
    image="python:3.12-slim",
    mode="ask",
    policy={
        "allowed_commands": ["python3", "pip", "cat", "ls"],
        "blocked_commands": ["curl", "wget", "nc", "rm -rf /"],
        "max_parallel": 4,
        "max_duration_seconds": 300,
    },
)
```


```typescript
import { LokaClient } from '@rizqme/loka-sdk';

const loka = new LokaClient();
const session = await loka.createSession({
    image: 'python:3.12-slim',
    mode: 'ask',
    policy: {
        allowedCommands: ['python3', 'pip', 'cat', 'ls'],
        blockedCommands: ['curl', 'wget', 'nc', 'rm -rf /'],
        maxParallel: 4,
        maxDurationSeconds: 300,
    },
});
```


## Update a policy at runtime



```python
client.set_policy(session.id, {
    "allowed_commands": ["python3", "pip", "cat", "ls", "git"],
    "blocked_commands": ["curl", "wget"],
})
```


```typescript
await loka.setPolicy(session.id, {
    allowedCommands: ['python3', 'pip', 'cat', 'ls', 'git'],
    blockedCommands: ['curl', 'wget'],
});
```



## Policy fields reference

| Field | Type | Description |
|---|---|---|
| `allowed_commands` | `string[]` | Binaries that always run without approval |
| `blocked_commands` | `string[]` | Binaries that are always rejected |
| `max_parallel` | `int` | Max concurrent commands per session |
| `max_duration_seconds` | `int` | Hard timeout for any single command |
