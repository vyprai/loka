# HA Mode

## Overview

HA mode runs 2+ control-plane instances. One is the **leader** (handles scheduling and state transitions); all instances serve API reads.

## Leader Election

Uses Redis `SETNX` with a TTL-based lease:

```
SETNX loka:leader 
EXPIRE loka:leader 15
```

The leader refreshes the lease every 5 seconds. If it fails to refresh, the key expires and another instance acquires leadership.

### Failover Behavior

1. Leader stops refreshing (crash, network partition).
2. Redis key expires after 15s TTL.
3. Remaining instances race to `SETNX`. Winner becomes leader.
4. New leader reconciles in-flight operations from PostgreSQL.
5. Workers reconnect to the new leader's gRPC stream.

Typical failover time: **15-20 seconds**.

## Distributed Locking

Per-resource locks prevent concurrent mutations:

```
SET loka:lock:session:  NX PX 10000
```

Operations that require locks: session create/destroy, checkpoint create/restore, worker drain.

## Pub/Sub Event Fanout

All instances subscribe to Redis pub/sub channels:

| Channel | Events |
|---|---|
| `loka:events:sessions` | created, started, paused, resumed, destroyed |
| `loka:events:commands` | submitted, approved, rejected, completed |
| `loka:events:workers` | registered, heartbeat, drained, removed |
| `loka:events:checkpoints` | created, restored, deleted |

This ensures all instances have a consistent view for API reads without polling PostgreSQL.

## Configuration

```yaml
ha:
  enabled: true
  instance_id: "cp-01"            # unique per instance
  leader_ttl_seconds: 15
  leader_refresh_seconds: 5
  lock_ttl_ms: 10000

postgres:
  dsn: "postgres://loka:pass@db:5432/loka?sslmode=require"
  max_connections: 20

redis:
  url: "redis://redis:6379/0"
  pool_size: 10

s3:
  bucket: "loka-state"
  endpoint: "https://s3.amazonaws.com"
  region: "us-east-1"
```

## Health Check

Each instance exposes `/healthz`:

```json
{
  "status": "ok",
  "is_leader": true,
  "leader_id": "cp-01",
  "pg_connected": true,
  "redis_connected": true,
  "uptime_s": 3600
}
```

Load balancers should route to instances where `status` is `ok`.
