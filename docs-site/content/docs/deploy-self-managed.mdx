# Self-Managed Workers

Connect your own servers to a LOKA control plane. Any Linux machine with KVM can be a worker.

## Setup

### 1. Create a registration token

On the control plane (or via CLI):

```bash
loka token create --name "my-server" --expires 86400
# Token: loka_a2d9fb8f2ff9d4f...
```

### 2. Install on your server

```bash
curl -fsSL https://rizqme.github.io/loka/install.sh | bash
```

### 3. Configure and start the worker

```bash
cat > /etc/loka/worker.yaml :9090"
data_dir: /var/loka/worker
provider: selfmanaged
token: "loka_a2d9fb8f2ff9d4f..."
labels:
  location: office
  gpu: "true"
YAML

loka-worker
```

### 4. Verify

```bash
loka worker list
# WORKER    HOSTNAME     PROVIDER      STATUS
# abc123    my-server    selfmanaged   ready
```

## Requirements

- Linux with KVM (`/dev/kvm`)
- Network access to the control plane on port 9090
- Docker (for image pulls)

## Labels

Workers can have labels for scheduling affinity:

```bash
loka worker label  gpu=true region=us-east
```

Sessions can target labeled workers:

```python
session = client.create_session(image="ubuntu:22.04", labels={"gpu": "true"})
```
