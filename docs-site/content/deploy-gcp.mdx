# Deploy to GCP

## Prerequisites

```bash
# Option 1: gcloud CLI (recommended)
gcloud auth login
gcloud config set project my-project-id

# Option 2: Service account key
export GOOGLE_APPLICATION_CREDENTIALS=/path/to/key.json

# Option 3: Pass directly
loka deploy gcp --project my-project-id --credentials /path/to/key.json
```

## Deploy

```bash
loka deploy gcp --project my-project --zone us-central1-a --workers 2
```

## Options

| Flag | Default | Description |
|------|---------|-------------|
| `--project` | | GCP project ID (required) |
| `--zone` | `us-central1-a` | Compute zone |
| `--workers` | `1` | Number of worker VMs |
| `--instance-type` | `n2-standard-8` | Worker machine type |
| `--credentials` | | Path to service account JSON |

## Nested virtualization

GCP requires nested virtualization to be enabled explicitly. `loka deploy gcp` handles this. For manual setup:

```bash
gcloud compute instances create loka-worker-1 \
  --zone us-central1-a \
  --machine-type n2-standard-8 \
  --min-cpu-platform "Intel Haswell" \
  --enable-nested-virtualization \
  --image-family ubuntu-2204-lts \
  --image-project ubuntu-os-cloud
```

`--enable-nested-virtualization` and `--min-cpu-platform "Intel Haswell"` are both required.

## Tear down

```bash
loka deploy destroy
```
