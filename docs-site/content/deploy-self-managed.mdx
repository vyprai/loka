# Self-Managed Workers

Connect any Linux machine with KVM to your LOKA control plane.

## Requirements

- Linux with `/dev/kvm`
- Network access to control plane on port 9090
- Docker (for image pulls)

## Setup

### 1. Start a control plane somewhere

```bash
# On any machine or cloud instance:
loka deploy local -f
```

### 2. Create a registration token

```bash
loka token create --name my-server --expires 86400
# Token: loka_a2d9fb8f...
```

### 3. Install on your server

```bash
ssh user@your-server
curl -fsSL https://rizqme.github.io/loka/install.sh | bash
```

### 4. Configure and start the worker

```bash
cat > /etc/loka/worker.yaml << YAML
control_plane:
  address: "<control-plane-host>:9090"
data_dir: /var/loka/worker
provider: selfmanaged
token: "loka_a2d9fb8f..."
labels:
  location: office
  gpu: "true"
YAML

loka-worker
```

### 5. Verify

```bash
loka worker list
```

## Labels

Workers can have labels for scheduling. Sessions can target workers with specific labels:

```bash
# Add labels
loka worker label <worker-id> gpu=true region=us-east

# Create session on a GPU worker
loka session create --image ubuntu:22.04 --labels gpu=true
```
